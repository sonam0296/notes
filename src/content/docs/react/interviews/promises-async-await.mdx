### Promise combinators

- `Promise.all([p1, p2, ...])` â†’ Runs all in parallel, resolves when all succeed, rejects if any one fails.
- `Promise.allSettled([p1, p2, ...])` â†’ Runs all in parallel, waits for all to finish, returns results (fulfilled or rejected).
- `Promise.any([p1, p2, ...])` â†’ Resolves as soon as any one succeeds, rejects only if all fail.
- `Promise.race([p1, p2, ...])` â†’ Resolves/rejects with the first promise that settles (success or failure).

- `async/await` is a **syntactic sugar** on top of Promises -> makes async code looks like synchronous code.
- Eg:

```

        try{
            const data = await getData()    // waits until resolved.
            console.log(data)
        }
        catch(err){
            console.log(err)
        }
    }

```

- âœ… Pros:
  - Much cleaner & more readable (looks like synchronous code).
  - Easier error handling with `try...catch`.
  - Great when you have multiple async calls in sequence.
- âŒ Cons:
  - You must be inside `async` function to use `await`.
  - If async calls can run in parallel, `await` might make them slower unless you use `Promise.all`.
